# 知识库前端代码审查修复计划

## Task 1: 安装 DOMPurify 并添加 XSS 净化（错误 #1，最高优先级）

**文件：** `src/components/WikiContent.vue`

1. 执行 `npm install dompurify` 安装依赖
2. 在 WikiContent.vue 中引入 DOMPurify：
   ```js
   import DOMPurify from 'dompurify'
   ```
3. 修改 `renderedHtml` computed，对 `md.render()` 输出做 sanitize：
   ```js
   const renderedHtml = computed(() => {
     if (!props.content) {
       return '<p class="empty-content">请从左侧选择一篇文档</p>'
     }
     try {
       const raw = md.render(props.content)
       return DOMPurify.sanitize(raw)
     } catch (e) {
       console.error('Markdown 渲染失败:', e)
       return '<p class="render-error">文档渲染失败，请检查文档格式</p>'
     }
   })
   ```
   此步同时解决了 **警告 #4（render 异常兜底）**。

## Task 2: 统一 TOC ID 与 anchorPlugin ID 生成逻辑（警告 #5）

**文件：** `src/utils/tocParser.js`

当前问题：`parseToc` 和 `anchorPlugin` 各自维护独立的 `idCount`，重复标题时生成的 ID 不一致，导致 TOC 点击无法跳转。

1. 抽取公共 ID 生成函数 `generateHeadingId(text, idCount)`
2. `parseToc` 和 `anchorPlugin` 都调用该公共函数，确保同一文档产生的 heading ID 完全一致
3. 修正 `parseToc` 中首次重复标题时 index 为 -1 的逻辑错误（L45: `idCount[baseId] - 1` 在第二次出现时为 0，实际应为 1）

## Task 5: anchorPlugin idCount 改用 env 参数（警告 #6）

**文件：** `src/utils/tocParser.js`

将 `anchorPlugin` 中闭包持有的 `idCount` 改为挂载在 markdown-it 的 `env` 参数上，使其跟随 render 调用的生命周期：

```js
md.renderer.rules.heading_open = (tokens, idx, options, env, self) => {
  if (!env.__idCount) env.__idCount = {}
  const idCount = env.__idCount
  // ... 后续逻辑使用局部 idCount
}
```

移除 `md.render` 的 monkey-patch 覆写。
