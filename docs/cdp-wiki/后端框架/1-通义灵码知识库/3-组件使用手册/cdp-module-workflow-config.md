# 如何使用 CDP 工作流 — 配置管理

## 概述

工作流配置管理包括流程模板、环节模板、变量模板、事件配置等基础配置，是流程设计的核心部分。

## 核心配置项

### 环节模板（Acttemplate）

定义流程中的审批环节：
- 环节名称和类型（审批、会签、知会等）
- 审批人规则（指定人、角色、部门负责人等）
- 时限设置和超时处理策略

### 变量模板（Vartemplate）

定义流程中可使用的变量，用于流程路由条件判断和表单数据传递。

### 事件配置（Event / Eventvar）

定义流程和环节级别的事件触发规则：
- 环节事件（Actevent）：环节开始、完成、退回时触发
- 互斥环节（Actmutex）：同一流程中互斥的环节配置
- 子流程（Actsubprocess / Callprocess）：主流程调用子流程
- 退回规则（Assignbackact）：定义可退回的目标环节

### 参与者配置

- **UserWf**：工作流用户配置
- **RoleWf**：工作流角色配置
- **Transactor**：办理人配置
- **UserAssignDepartment**：用户部门分配

### 标签与分类

- **Tag / Tagdetail**：流程标签管理
- **CommentCategory / CommentGroup**：审批意见分类管理

### 任务模板

- **Jobtemplate / Jobtemplatefile**：定时任务模板
- **JobLog**：任务执行日志

### 静态字典

- **StaticDict**：工作流内部使用的静态字典数据

## 配置文件

### ot-tree.xml

组织树配置文件，定义工作流中使用的组织结构数据来源。

### applicationContext-wfevent.xml

事件监听器和消息发送配置（详见流程定义文档）。

## 注意事项

> 注意：流程模板修改后，已运行的流程实例不受影响，仅新发起的流程使用新模板。

> 注意：环节的审批人规则支持多种策略，可组合使用（如：先按角色匹配，无人时使用部门负责人）。

> 注意：子流程（Callprocess）可实现流程复用，主流程等待子流程完成后继续执行。
