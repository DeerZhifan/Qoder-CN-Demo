# 如何使用 CDP 工作流 — 任务管理与审批

## 概述

工作流任务管理处理审批任务的接收、处理、转办和委托等操作，是用户与工作流引擎交互的主要入口。

## 核心功能

### 任务操作

- **待办任务**：用户登录后查看分配给自己的待审批任务
- **任务处理**：审批通过、退回、加签、转办等操作
- **任务委托（TaskSurrogate）**：将任务委托给其他用户处理
- **催办（Urging）**：对超时未处理的任务发送催办提醒
- **传阅（Circulat）**：将流程信息传阅给相关人员

### 任务查询

- 按用户查询待办任务
- 按流程实例查询任务历史
- 按部门/组织查询任务统计

### 审批意见

- **Opinion**：记录每个环节的审批意见
- 支持分类标签和评论组管理

## 业务开发集成

业务系统通过监听流程事件来处理审批结果：

```java
// 业务单据状态更新示例
// 当流程审批通过时，更新业务单据状态
public void onProcessCompleted(String billId) {
    // 更新业务单据状态为"已审批"
    billService.updateStatus(billId, "APPROVED");
}
```

## 注意事项

> 注意：任务委托（TaskSurrogate）可设置委托时间范围，在指定时间段内自动将任务转给委托人。

> 注意：催办功能需配置消息发送方式（短信/微信等），在 `applicationContext-wfevent.xml` 中配置。

> 注意：传阅操作仅通知，不需要传阅人进行审批操作。
